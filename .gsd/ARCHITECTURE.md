## Data Flow

### Live Demo Pipeline Flow
1. **User Input** → Text typed or selected from example chips (Punjabi/Hindi/Romanized)
2. **STT Stage** → Simulated (text input acts as transcribed audio) — 600ms delay
3. **Guardrails Stage** → Anthropic Claude API call classifies `{is_farming, is_safe, reason}`
   - Falls back to keyword matching if API fails
4. **Decision Gate:**
   - `is_safe=false` → BLOCKED
   - `is_farming=false` → REDIRECT with Punjabi message
   - Both true → ALLOWED → continue
5. **Intent Router** → Local regex classification into 6 categories
6. **Response Generator** → Anthropic Claude API generates advice in source language
   - Falls back to hardcoded Punjabi responses per intent
7. **UI Update** → JSON panel, decision badge, response panel with meta tags

### Phone Call Flow
1. **Call Button** → Toggles call state
2. **Mic Access** → Web Speech API (`pa-IN` locale)
3. **Speech Recognition** → Transcript displayed as chat bubble
4. **Pipeline** → Same guardrails → router → response flow
5. **TTS Output** → Browser SpeechSynthesis speaks response in Punjabi

## Technical Debt

- [ ] **Monolithic HTML** — 2,347-line single file contains all CSS, HTML, and JS. Should be split into separate files.
- [ ] **No backend implementation** — `Backend/` directory is empty. All AI pipeline is simulated in the frontend.
- [ ] **API key exposure risk** — Anthropic API calls made directly from frontend JS (no proxy/backend).
- [ ] **No build system** — Static HTML served directly, no bundling, minification, or dev tooling.
- [ ] **Hardcoded fallback responses** — Intent-specific responses are hardcoded strings, not from a knowledge base.
- [ ] **No real ML pipeline** — The Whisper, Phi-3, gTTS models are described in architecture but not implemented. Demo uses Claude API as proxy.
- [ ] **No tests** — Zero test coverage.
- [ ] **No error handling UI** — API failures silently fall back to heuristics with no user feedback.
- [ ] **Regex-based intent detection** — `detectIntent()` uses simple regex, not a trained model.
- [ ] **No responsive testing** — Media queries exist but limited to 900px breakpoint.
- [ ] **No accessibility** — Missing ARIA labels, keyboard navigation, screen reader support.
